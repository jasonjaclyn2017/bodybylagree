Pixel-perfect reproduction checklist

Details that aren't obvious from a glance at the code. Most of the layout is plain CSS grid + breakpoints, but these few things carry the look and are easy to get subtly wrong when rebuilding on Framer.

Frame & shape

The hexagon points are a fixed 50px (calc(100% − 50px)), not a percentage — they must stay 50px no matter how tall the section grows. A stretched SVG would distort them.
The frame is 4 stacked clip-path layers (insets 0 / 1.5 / 7 / 8.5px; cream → black → translucent cream → black) — that's what makes the hairline + inner double-line. If rebuilding as SVG, use one polygon with two strokes.
Section shadow is a 3-layer drop-shadow including a negative-Y (top) shadow that lifts the section above the hero video. Don't drop it.

Typography

"Built with Intention" auto-fits to the prose column width via container units clamp(24px, 9.6cqi, 36px) + nowrap; the column is container-type: inline-size. It is not viewport-sized — sizing to vw will be wrong.
The big "THE BBL SOCIETY EXPERIENCE" is an image asset (contrast baked in for the black bg), not live text — it scales via width:100%.
Font is Manrope: weight 300 for the display line, 600 for eyebrow/card titles, 700 for the offer.

Interaction

Hovering one image dims the other two (:has() sibling rule), not a self-hover. On Framer this needs the mosaic to be one component holding shared hover state — variants alone can't reach siblings.
Hovered cell scales 1.05 and lifts via z-index so it overlaps neighbors; the inner image also scales 1.03 on a separate, slower curve. No per-cell hover box-shadow (intentionally removed) — the mosaic's only shadow is the static container shadow noted under Layout.
Chevron is a 2-line polyline (~140°), not an icon font. On hover: opacity .4→1, color #D2CDC2→#eee, slides +8px right. Title underline grows 0→100%.

Placement

The hexagon's top point pokes ~30px above the hero-video bottom edge (the dashed line) — a fixed overlap, no scroll trigger, no transition. Section top sits 30px above that line at every width (coupled via calc()).
The negative-Y drop-shadow reinforces the lift over the hero. Nothing below the section moves, so there are no layout gaps to chase.

Layout & responsive

Image heights come only from aspect-ratio (feature 19/12, bands 19/8) at every width — never hardcode px heights, or you'll reintroduce the collapse-at-breakpoint bug.
Bottom padding (104px) is load-bearing: it keeps a hover-popped image inside the 50px bottom point. Don't go below ~62px.
Breakpoints: 2-col 40/60 (2fr 3fr) ≥806px → 1-col <806px; corner ticks hidden <806px; cream side margin crushes to 2px ≤480px. Columns are top-aligned (align-items: start).
The 3px gaps between images are real — the black (#0D0C0B) mosaic background shows through as thin seam lines.
The mosaic block sits on a static offset shadow (box-shadow: 6px 6px 30px -4px #2c2c2c) that grounds it against the black interior — this is separate from the (intentionally absent) per-cell hover shadow.

Tokens

#FFFBF2 page · #0D0C0B frame · #D2CDC2 cream accent · #FBF7EE near-white text · #B8B2A5 body · #C75D4A spec/annotation accent.
Each image carries a bottom-up dark scrim gradient for label legibility (slightly stronger on the two bands than the feature). Feature image uses object-position: center 30%.
